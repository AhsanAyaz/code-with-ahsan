# Quick Task 260609-f6n — Participants on challenge detail + cert data snapshot

Two related asks from Maham:
1. Once a user participates, show them on the leaderboard **with submission status**.
2. Capture contact info (email / discord) for later certificate sharing — **without a
   form** (derive from the user record at join time).

## Decisions (locked)
- Placement: a **Participants** section on the challenge **detail page**
  (`/challenges/[id]`), next to the existing count + mini leaderboard.
- Capture: snapshot `email` (+ `discordUsername` when available) onto the
  participant doc **at join**. No form. Public list shows avatar + name + status
  only — **email/discord never rendered publicly** (admin-only export later).

## Tasks

### Task 1 — Capture contact info at join (data layer)
- `src/types/challenges.ts`: add optional `email?` and `discordUsername?` to
  `ChallengeParticipant`. Add public-safe `ChallengeParticipantStatus` interface
  (`userId, userName, userAvatar?, joinedAt, submitted, submittedAt?`).
- `src/lib/challengeProfiles.ts`: extend `getChallengeParticipantProfile` to also
  return `email` and `discordUsername`. Read `users` and, for discord/email gaps,
  `mentorship_profiles`; fall back to firebase auth `email`.
- `src/services/ChallengeService.ts`: `normalizeParticipant` returns the new
  fields; `joinChallenge` accepts + stores `email`/`discordUsername`, stripping
  undefined before the Firestore write (same undefined-rejection guard as
  `userAvatar`/`demoUrl`).
- `src/app/api/challenges/[id]/participants/route.ts` (POST): pass `email` +
  `discordUsername` from the profile into `joinChallenge`.

### Task 2 — Participants list with submission status (UI)
- `src/services/ChallengeService.ts`: add
  `getChallengeParticipantsWithStatus(challengeId)` — load participants +
  submissions, return the public-safe `ChallengeParticipantStatus[]` (no
  email/discord), submitted entries first then by `joinedAt`.
- `src/app/challenges/[id]/page.tsx`: add a Participants section (SSR) rendering
  avatar + name + a badge: **Submitted ✓** (success) / **Joined — not submitted**
  (ghost). Empty state when none.

## Verify
- `npx tsc --noEmit` / lint clean
- E2E (emulators): join a user → appears in participants-with-status as
  not-submitted; submit → flips to submitted. Participant doc carries email
  (+discord when present); status payload omits email.
