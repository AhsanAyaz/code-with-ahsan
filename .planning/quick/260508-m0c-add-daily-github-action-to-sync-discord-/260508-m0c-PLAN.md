Implementation details:

a. Add a 5th element to the existing Promise.all so the parallel fan-out becomes [mentors, mentees, sessions, ratings, discordStats]:
```ts
const [mentorsSnapshot, menteesSnapshot, sessionsSnapshot, ratingsSnapshot, discordStatsSnap] = await Promise.all([
  db.collection('mentorship_profiles').where('role', '==', 'mentor').where('status', '==', 'accepted').get(),
  db.collection('mentorship_profiles').where('role', '==', 'mentee').get(),
  db.collection('mentorship_sessions').get(),
  db.collection('mentor_ratings').get(),
  db.collection('platform_stats').doc('discord').get(),
]);
```

b. After computing community fields, derive the merged social map:
```ts
let mergedSocial: typeof socialReach = socialReach;
try {
  if (discordStatsSnap.exists) {
    const data = discordStatsSnap.data();
    const liveCount = data?.memberCount;
    if (typeof liveCount === 'number' && Number.isFinite(liveCount) && liveCount > 0) {
      mergedSocial = {
        ...socialReach,
        discord: { ...socialReach.discord, count: liveCount },
      };
    }
  }
} catch (e) {
  console.error('platform_stats/discord merge failed, using static fallback:', e);
}
```

c. Use `mergedSocial` in the `statsCache.social` assignment instead of `socialReach`.

Important constraints:
- A failure of the Firestore platform_stats fetch must NOT break the API. Because the doc fetch lives inside the existing top-level try/catch via Promise.all, an outright reject would surface as a 500. To honor the fallback contract, wrap the platform_stats fetch separately so its failure degrades to the static value:

Replace step (a) with this safer shape:
```ts
const platformStatsPromise = db.collection('platform_stats').doc('discord').get()
  .catch((err) => {
    console.error('platform_stats/discord fetch failed, using static fallback:', err);
    return null;
  });

const [mentorsSnapshot, menteesSnapshot, sessionsSnapshot, ratingsSnapshot, discordStatsSnap] = await Promise.all([
  db.collection('mentorship_profiles').where('role', '==', 'mentor').where('status', '==', 'accepted').get(),
  db.collection('mentorship_profiles').where('role', '==', 'mentee').get(),
  db.collection('mentorship_sessions').get(),
  db.collection('mentor_ratings').get(),
  platformStatsPromise,
]);
```

Then guard step (b) with `if (discordStatsSnap && discordStatsSnap.exists) { ... }`.

This keeps the response shape identical (CachedStats type unchanged — `social: typeof socialReach` already covers the merged map).
