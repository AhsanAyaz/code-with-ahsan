## What Was Built

### API Routes

- **POST /api/mas-raffle/entries** — public, validates name/email/newsletter, writes `mas-raffle-emails/{autoId}` with `date` field for daily scoping
- **GET /api/mas-raffle/entries/count** — admin-gated, returns `{ count }` for today using AggregateQuery
- **POST /api/mas-raffle/spin** — admin-gated, three actions:
  - `spin`: picks random winner from today's pool, writes `mas-raffle-state/current { state: "spinning" }`, returns `{ winnerName, docId }`
  - `confirm`: writes `state: "winner"`, deletes winner doc from pool (after 4s client delay)
  - `reset`: writes `state: "idle"` to allow next spin

### Public Page (`/mas-raffle`)

Four UI states driven by Firestore onSnapshot + localStorage:

1. **Form** — DaisyUI card with name/email inputs and newsletter checkbox; POST on submit; localStorage key set on success
2. **Waiting** — pulsing gradient ring, "Entry submitted" badge; shown immediately on revisit if localStorage key present
3. **Spinning** — dual-ring CSS spinner (inner `animate-spin`, outer reverse via inline style `animation: spin 1.5s linear infinite reverse`)
4. **Winner** — trophy emoji, large winner name with `popIn` keyframe animation, canvas-confetti burst

Admin reset transitions "winner"/"spinning" viewers back to "waiting" (if already submitted) or "form".

### Admin Page (`/admin/mas-raffle`)

Wrapped in `AdminAuthGate` — password prompt before content is shown.

- Today's entry count with refresh link (AggregateQuery via admin-gated endpoint)
- State badge (idle=neutral, spinning=warning, winner=success) synced via onSnapshot
- "Spin the Wheel" button — disabled when no entries, already winner, or mid-spin
- 4s timeout between spin and confirm — public viewers see the animation, then winner is revealed
- Winner reveal card with name in large success text
- "Reset for next spin" button after winner is shown

### Firestore Rules

Two new blocks appended inside `match /databases/{database}/documents`:

```
/mas-raffle-emails/{entryId}
  allow create: field validation (hasAll keys + type checks)
  allow read, update, delete: if false   ← T-g81-04

/mas-raffle-state/{docId}
  allow read: if docId == "current"      ← public onSnapshot
  allow write: if false                  ← T-g81-03, Admin SDK only
```

## Deviations from Plan

### Auto-added: entries/count as a sub-route

The plan described the count endpoint inline in `entries/route.ts`. To avoid conflating a public POST handler with an admin GET handler in the same file, the count was moved to `src/app/api/mas-raffle/entries/count/route.ts` — a clean Next.js sub-route. The client call `GET /api/mas-raffle/entries/count` is unchanged.
