**`src/app/api/mas-raffle/entries/route.ts` — POST (public, no auth)**

Write to `mas-raffle-emails` collection with auto-id:
```

```
Return `{ ok: true }` on success, `{ error: "..." }` with 400/500 on failure.

**`src/app/api/mas-raffle/spin/route.ts` — POST (admin-gated) and POST /reset (admin-gated)**

Use `requireAdmin` (import from `@/lib/apiAuth` — check existing exports; if not available use the same inline token check pattern: `const token = req.headers.get("x-admin-token"); const valid = await verifyAdminToken(token)`).

Actually: look at how existing admin routes verify the token. Check `src/app/api/mentorship/admin/auth/route.ts` to understand how requireAdmin works, then import it.

POST (spin):
1. Query `mas-raffle-emails` where `date == today` (server UTC date).
2. If no entries: return `{ error: "No entries for today" }` with 400.
3. Pick random entry: `entries[Math.floor(Math.random() * entries.length)]`.
4. Write `mas-raffle-state/current` with `{ state: "spinning", winnerName: winner.name, date: today, updatedAt: FieldValue.serverTimestamp() }`.
5. After writing "spinning", schedule a write of `{ state: "winner", ... }` by immediately returning the response — the client (AdminRaffleClient) will call a second endpoint after the animation delay. Actually: write both states server-side with a delay is not feasible. Instead:
   - POST /spin writes `state: "spinning"` and returns `{ winnerName }`.
   - Admin client waits 4000ms then calls POST /spin/confirm (or include action param).
   - Simpler: POST body `{ action: "spin" }` → writes "spinning" + returns winner. POST body `{ action: "confirm", winnerName }` → writes "winner" + deletes winner doc from pool.
6. On action "confirm": set `mas-raffle-state/current` to `{ state: "winner", winnerName, date, updatedAt }`. Then query `mas-raffle-emails` where `email == winnerEmail` AND `date == today` to find and delete winner doc. Return `{ ok: true }`.

POST (reset — use body `{ action: "reset" }`):
- Write `mas-raffle-state/current` with `{ state: "idle", winnerName: null, date: today, updatedAt: FieldValue.serverTimestamp() }`.
- Return `{ ok: true }`.

Note on winner deletion: The spin action returns `winnerName` but to delete we need the doc ref. Capture `winner.id` (the doc ID) on spin action and return it. Client sends `{ action: "confirm", docId: winner.id, winnerName: winner.name }` after animation delay. Server deletes `mas-raffle-emails/{docId}` and writes "winner" state.
