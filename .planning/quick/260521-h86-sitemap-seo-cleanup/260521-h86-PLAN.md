Implementation structure (do NOT inline as code in this plan — write it in the file):

1. Define `const BASE_URL = 'https://codewithahsan.dev'` at module top.
2. Define `const now = new Date()` for default `lastModified` when no per-entry date exists.
3. Build the entries array in this order:
   a. Homepage entry: `{ url: BASE_URL, lastModified: now, changeFrequency: 'daily', priority: 1.0 }`
   b. Static-page entries — iterate over a typed const array of paths (see `<sitemap_inventory>` PUBLIC STATIC ROUTES, EXCLUDING homepage). Each gets `priority: 0.7`, `changeFrequency: 'monthly'`, `lastModified: now`.
   c. Dynamic event entries — `await getEvents()` then map each to `{ url: \`${BASE_URL}/events/${event.slug}\`, ... }` with `priority: 0.7`, `changeFrequency: 'monthly'`. Use `event.updatedAt ?? event.publishedAt ?? now` for `lastModified` (cast to `new Date()` if string).
   d. Course detail entries — `await getCourses()` then for each course emit one entry at `priority: 0.8, changeFrequency: 'weekly'`. Use `course.updatedAt ?? course.publishedAt ?? now`.
   e. Course resources entries — for each course emit `/courses/${course.slug}/resources` at `priority: 0.3, changeFrequency: 'weekly'`.
   f. Course submissions entries — for each course emit `/courses/${course.slug}/submissions` at `priority: 0.3, changeFrequency: 'weekly'`.
   g. Course post entries — for each course → each chapter → each post emit `/courses/${course.slug}/${post.slug}` at `priority: 0.6, changeFrequency: 'weekly'`. Use `post.updatedAt ?? post.publishedAt ?? course.updatedAt ?? now`.
4. Return the assembled array. Use `Array.flatMap` to keep the body compact.
5. Guard against undefined: chapters or posts may be missing → use `(course.chapters ?? []).flatMap(ch => ch.posts ?? [])`.

Imports to add at the top:

- `import { getCourses, getEvents } from '@/lib/content/contentProvider';`

Type the static path list as `readonly string[]` or `const staticPaths = [...] as const` to keep it tight.
