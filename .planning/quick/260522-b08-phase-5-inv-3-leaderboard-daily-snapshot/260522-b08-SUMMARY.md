## Architectural Decision Record

**Decision:** Daily leaderboard snapshot at 07:00 UTC (was hourly per original DASH-07 spec).

**Locked:** 2026-05-22 by user, before this quick task ran.

**Rationale:** "Small ambassador cohort; daily cadence is sufficient. Hourly was over-engineered for current scale." (from quick-task frontmatter)

**Implications:**
- Snapshot doc freshness ≤ 24h on the read path. UI label adapts via `formatUpdatedAgo` (`min` / `h` / `d` branches).
- Live-compute fallback in the route handles the ≤ 24h cold-start window before the first cron tick lands. Cost is bounded (single-cohort fan-out) and identical to the previous in-memory-cache cold start.
- Can be raised to hourly later by changing exactly one line (`'0 7 * * *'` → `'0 * * * *'`) in `.github/workflows/ambassador-activity-checks.yml` plus the equivalent header comment in the script. No app code changes required.
