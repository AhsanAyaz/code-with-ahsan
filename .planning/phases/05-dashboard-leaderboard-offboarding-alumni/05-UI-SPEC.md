## Spacing Scale

Declared values (must be multiples of 4). Inherited from Phase 4 UI-SPEC — no new exceptions for Phase 5.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline label gaps (`gap-1`) |
| sm | 8px | Compact element spacing, stat item internals (`gap-2`) |
| md | 16px | Default element spacing, card body gaps (`gap-4`) |
| lg | 24px | Section padding, card body outer (`gap-6`) |
| xl | 32px | Layout gaps between major dashboard sections (`gap-8`) |
| 2xl | 48px | Major section breaks (`space-y-12`) |
| 3xl | 64px | Page-level spacing — used for dashboard layout top/bottom padding |

Exceptions:
- Touch targets for mobile offboard / alumni confirm buttons: minimum 44px height — use `btn btn-sm min-h-[44px]` on mobile viewports
- Loading spinner containers: `py-12` (48px) vertical padding — matches existing pattern in MemberDetailClient and ReportPageClient
- Leaderboard category columns on narrow screens: allow `gap-4` horizontal collapse to `gap-2` at `sm` breakpoint

## Typography

Source: Rubik font stack confirmed in tailwind.config.js and globals.css. Identical to Phase 4 — no new type sizes introduced.

| Role | Size | Weight | Line Height | DaisyUI Class |
|------|------|--------|-------------|---------------|
| Body | 16px | 400 (regular) | 1.5 | base prose or none |
| Label | 14px | 700 (bold) | 1.4 | `label-text font-bold` |
| Heading | 20px | 700 (bold) | 1.2 | `card-title` / `text-xl font-bold` |
| Display | 28px | 700 (bold) | 1.2 | `text-2xl font-bold` (page h1 headings) |

Maximum 2 weights per surface: regular (400) for body/help text, bold (700) for labels, headings, and card-titles. Semibold (600) is not used in Phase 5.

Stat values in the personal stats panel and leaderboard use `stat-value` (DaisyUI — renders at ~30px, bold). This is an intentional exception to the 4-size cap because `stat` is a DaisyUI composition component, not a new type role.

## Color

Source: globals.css `:root` overrides. DaisyUI semantic tokens are the contract — do not use raw hex values in new Phase 5 components.

| Role | Value | DaisyUI Token | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | base-100 (light/dark) | `bg-base-100` | Dashboard page background, card surfaces |
| Secondary (30%) | base-200 / base-300 | `bg-base-200`, `bg-base-300` | Admin panel table rows, leaderboard row alternation, filter bars |
| Primary accent (10%) | #8f27e0 (purple) | `text-primary`, `btn-primary`, `badge-primary` | Primary CTAs, active tab/view indicator, personal rank highlight, Ambassador badge |
| Secondary accent | #d926a9 (pink) | `badge-secondary` | Alumni Ambassador badge only (matches AmbassadorBadge `badge-secondary`) |
| Success | DaisyUI success | `badge-success`, `alert-success` | Onboarding item completed state, on-time report badge |
| Warning | DaisyUI warning | `badge-warning`, `alert-warning` | Leaderboard grace-period banner, 2-strike warning state |
| Error / Destructive | DaisyUI error | `alert-error`, `btn-error` | Offboard confirm button, Discord-removal retry failure banner, strike count at 2 |
| Info | DaisyUI info | `alert-info`, `badge-info` | Empty states, "Updated N minutes ago" staleness notice, leaderboard-locked banner |

Accent (primary) reserved for:
- Primary CTA buttons (Refresh leaderboard, Mark done for onboarding items)
- Personal rank highlight row in leaderboard ("Your rank: #N" label)
- Active view toggle tab (Cumulative vs This Month)

Destructive (error) reserved for:
- Offboard confirmation button inside OffboardConfirmModal
- Discord role removal retry failure banner
- Strike count display when strikes === 2 (already established in ActivitySummaryPanel)

Do NOT use `btn-primary` for the Alumni transition button — it is an admin-facing action on the member detail page. Use `btn-outline` with `btn-accent` or `btn-ghost` to visually distinguish it from the destructive offboard button.

## Interaction States

Every interactive element must implement all six states. Inherited from Phase 4 — no new states for Phase 5.

| State | Implementation |
|-------|---------------|
| Default | DaisyUI component default |
| Loading | `loading loading-spinner loading-sm` inside button; button `disabled` |
| Success | `toast.success(...)` via ToastContext |
| Error | `alert alert-error` inline OR `toast.error(...)` for non-blocking errors |
| Empty | `alert alert-info` with descriptive copy + next-step link |
| Disabled | `disabled` attribute; DaisyUI applies `opacity-50` automatically |

Additional Phase 5 states:

| State | Implementation |
|-------|---------------|
| Leaderboard locked (grace period) | `alert alert-info` banner inside `LeaderboardPanel` with "Leaderboard unlocks in N weeks" copy; no table rendered |
| Onboarding item completed | Checkbox checked state + `line-through text-base-content/50` on item label |
| Discord removal failure (offboarding) | `alert alert-error` inline in MemberDetailClient after POST `/offboard`; shows "Discord role removal failed — retry below" with a `btn btn-outline btn-sm` retry |
| Alumni transition (term-end) | `btn btn-outline` with confirmation prompt — NOT a modal (less consequential than strike offboarding) |

## Visual Hierarchy Notes

### `/ambassadors/dashboard`

Layout top-to-bottom order:
1. Page heading + subheading (display text)
2. `AmbassadorOfMonthBanner` — only renders when field is set on cohort doc; if unset, skip (never render a blank banner)
3. `OnboardingChecklist` — first-visit priority surface; render it prominently above stats until all 5 items are complete; after all items complete, collapse to a single "All done" card
4. `PersonalStatsPanel` — always visible; 4-column stat grid on desktop, 2-column on mobile
5. `LeaderboardPanel` — bottom of page; grace-period banner or leaderboard table; never a focal point during grace period

Focus order intent: New ambassadors land on the checklist. Experienced ambassadors scan their stats, then optionally scroll to the leaderboard.

### `/admin/ambassadors/members/[uid]` — Phase 5 additions

- `AlumniTransitionButton` renders below the existing `StrikePanel` section, separated by a `divider`
- `OffboardConfirmModal` is triggered from the existing "Remove from program" button area — replace the existing `window.confirm()` call with `OffboardConfirmModal` (Phase 4 research identified this as a deprecated pattern)
- Discord removal failure banner renders inline below the modal close, NOT as a toast (admin must see it persistently to act on it)
- Section order: ActivitySummaryPanel → CronFlagsPanel → StrikePanel → (divider) → Alumni / Offboard actions → Report History → Event History

### Leaderboard table layout

- 3 columns, one per category (Referrals | Events Hosted | Reports On Time)
- Each column shows: rank 1, rank 2, rank 3 with avatar/name and count
- "Your rank: #{N}" displays below the 3-column table as a full-width row with `bg-primary/10` background to create private visibility without breaking the top-3 layout
- Do NOT show the full sorted list — only top-3 per category + own rank row

## Accessibility Constraints

Inherited from Phase 4 — all constraints apply. Phase 5 additions:

- `OffboardConfirmModal` must use `<dialog className="modal modal-open">` with `aria-labelledby` pointing to the modal heading — same pattern as `StrikeConfirmModal`
- `OffboardConfirmModal` heading must include `{displayName}` — prevents admin offboarding wrong ambassador
- Leaderboard table must use `<table>` semantics (not CSS grid) so screen readers can navigate column headers
- Leaderboard "Your rank" row must have `aria-label="Your rank in the {category} category"` per column
- Onboarding checklist items: use `<ul role="list">` with each item as `<li>`; self-mark items use `<button type="button">` (not `<input type="checkbox">`) for explicit user intent signaling
- "Updated N minutes ago" text must have `aria-live="polite"` so screen readers announce it after a manual refresh
- Alumni transition inline confirmation: use `role="alert"` on the confirmation prompt so it is announced when it appears
