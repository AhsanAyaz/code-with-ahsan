<objective>
**Rationale (CONTEXT.md §AGENT-STATE-01 + RESEARCH §2.3 lines 305-308):** ADK's `output_key` is *scalar* — one LlmAgent writes exactly one key (RESEARCH §2.3 lines 305-308). To get BOTH `user_skill_level` and `user_goals` into session.state, the planner had two choices: (Option A) a Pydantic `output_schema` on a single LlmAgent + a downstream unpacker step, OR (Option B) split onboarding into two LlmAgents under a SequentialAgent. CONTEXT.md §AGENT-STATE-01 locks Option B ("two top-level keys, plain strings, no Pydantic wrapping") — chosen because (1) downstream templating reads `{user_skill_level?}` / `{user_goals?}` directly without an unpacker round-trip, (2) ADK's `{key?}` templating does NOT support attribute access (RESEARCH §5.2 pitfall), making Option A's `{onboarding?.skill_level}` form non-viable without extra code, and (3) two atomic LLM calls (each ≤200 tokens output) is cheaper + more debuggable in `adk web`'s Events tab than one larger structured-output call. This plan implements Option B.

Wire the `output_key` whiteboard between onboarding and the three downstream agents (mentorship, projects, roadmap). Replace the existing single-LlmAgent `onboarding_agent` with a `SequentialAgent` composing three children — `skill_level_extractor` (output_key="user_skill_level"), `goals_extractor` (output_key="user_goals"), and `welcome_agent` (the existing user-facing reply, no output_key, runs LAST so the SequentialAgent surfaces its warm welcome as the final user-visible response per RESEARCH Open Question 2 recommendation, lines 826-830).

Purpose: Closes AGENT-STATE-01 (writes) and AGENT-STATE-02 (reads). Establishes the `output_key` namespace lock from `.planning/milestones/v7.0-ROADMAP.md ## Cross-Phase Contracts` (`user_skill_level`, `user_goals`) that Phase 8's `mentor_intake_clarifier` will extend (`user_timezone`, `user_focus_area`, `next_clarifying_question`).

Output: Refactored `onboarding_agent.py` (SequentialAgent topology), three downstream agents with a USER CONTEXT instruction block prepended (RESEARCH §2.4 exact before/after for `mentorship_agent`, same pattern for projects + roadmap), a new pytest file with topology + substring assertions, and a documented smoke-test step in `adk web` to verify the SequentialAgent child order surfaces the welcome reply correctly (RESEARCH Open Question 2 — the only critical gate this plan owns).
