# Phase 02.1 Plan 02: YouTube Search Route + content_agent Extension Summary

YouTube ISR-cached Next.js proxy route scoped to Code With Ahsan's channel + extended content_agent exposing both blog and YouTube search tools with intent-based routing strategy.

## What Was Built

**Task 1 — /api/content/youtube/search route (Next.js)**

- `src/app/api/content/youtube/search/route.ts`: GET handler that proxies YouTube Data API v3 `/search` with `next: { revalidate: 3600, tags: ["content:youtube"] }` ISR cache directive. Scoped to `CWA_CHANNEL_ID = "UCAys-Lg76QcRNGc0dOr_bXA"` (Code With Ahsan's channel) — prevents arbitrary YouTube results leaking through. Maps each item to `{ videoId, title, description (truncated 300), url, thumbnail, published_at, channel_title }`. Handles all error paths: missing q → 400, unconfigured key → 500, upstream non-2xx → 502, fetch throw → 500. Uses `YT_API_KEY` (not `YOUTUBE_API_KEY`) consistent with v4.0 admin route. Does NOT log the API key.
- `src/app/api/content/youtube/search/__tests__/route.test.ts`: 5 Vitest cases using `vi.stubGlobal("fetch", ...)` with `vi.unstubAllGlobals()` in `beforeEach`. Covers happy path (asserts `videos[0].url`, `count`, fetch mock called with `channelId=UC...` and `next.revalidate===3600`), 400, 500 (no key), 502, and 500 (throw).

**Task 2 — content_agent + platform_client extension + pytest tests**

- `agent/community_assistant/platform_client.py`: Added `fetch_youtube_videos(query)` directly after `fetch_blog_posts`. Single-line `_get("/api/content/youtube/search", {"q": query})` call. One-line docstring matching existing helper style.
- `agent/community_assistant/sub_agents/content_agent.py`: Extended (not recreated). Added `_YOUTUBE_LIMIT = 5`, `_shape_youtube_video(raw)` with url passthrough/fallback/None logic, `search_youtube_videos(query)` tool with empty-query guard and error contract `{"status": "error", "videos": []}`. Both `search_blog_posts` and `search_youtube_videos` registered in `tools=` list. Updated `description=` to mention YouTube. Updated `instruction=` with TOOLS section listing both tools, SEARCH STRATEGY routing by intent (VIDEO/YouTube→YT, BLOG/ARTICLE→blog, ambiguous→both), and GUIDELINES with URL citation mandate.
- `agent/tests/fixtures/content_youtube.json`: 3 realistic CWA channel videos (Angular Signals, RxJS, NgRx) with all route response fields.
- `agent/tests/conftest.py`: Added `content_youtube_payload` fixture.
- `agent/tests/test_content_agent_youtube.py`: 7 pytest cases covering shape url passthrough, url fallback from videoId, None url, description truncation, integration success (all URLs start with youtube.com/watch?v=), empty query error, and error path.

## Decisions Made

1. **Channel ID resolution**: Used `curl` on the public `youtube.com/@codewithahsan` page to extract the `UC`-prefixed channel ID — no API key required, no code in other files had it hard-coded.
2. **Direct fetch vs googleapis SDK**: Direct `fetch()` is required to pass `{ next: { revalidate: 3600 } }` — the googleapis SDK does not surface Next.js ISR cache directives.
3. **YT_API_KEY env var name**: Reused existing `YT_API_KEY` (not `YOUTUBE_API_KEY`) to avoid duplicate env var; the admin courses route already uses this name.
4. **Intent-based routing in instruction**: Three-way strategy (VIDEO/YouTube intent → YT tool, BLOG/ARTICLE intent → blog tool, ambiguous → both) prevents the agent from defaulting to one tool and missing content.
5. **No root agent re-wiring**: `content_agent` was already registered in `agent.py` by Plan 01 — only `content_agent.py` was extended.
