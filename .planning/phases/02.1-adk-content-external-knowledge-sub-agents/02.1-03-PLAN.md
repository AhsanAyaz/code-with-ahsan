Upstream API contracts:

**GitHub** — `GET https://api.github.com/search/repositories?q={q}&sort=stars&order=desc&per_page=5`
- Headers: `Accept: application/vnd.github+json`, `X-GitHub-Api-Version: 2022-11-28`, optional `Authorization: Bearer {GITHUB_TOKEN}` (raises quota 60/hr → 5000/hr)
- Response: `{ total_count, items: [{ full_name, html_url, description, stargazers_count, language, updated_at, topics }] }`
- Map to: `{ name: full_name, url: html_url, description, stars: stargazers_count, language, updated_at, topics }`

**dev.to** — `GET https://dev.to/api/articles?per_page=5&tag={q}` (preferred — tag-scoped search)
- Fallback when tag returns 0: `GET https://dev.to/api/articles?per_page=5&top=7` (top of last 7 days, then post-filter by title/tag-list substring match)
- No auth required for basic search
- Response: array of `{ title, description, url, cover_image, tag_list, public_reactions_count, published_at, user: { name, username } }`
- Map to: `{ title, url, description, tags: tag_list, reactions: public_reactions_count, published_at, author: user.name }`

**Stack Overflow** — `GET https://api.stackexchange.com/2.3/search/advanced?order=desc&sort=votes&q={q}&site=stackoverflow&pagesize=5&filter=withbody&answers=1`
- No auth required for read; `answers=1` filter for only-answered questions
- Response: `{ items: [{ title, link, score, answer_count, is_answered, tags, creation_date, owner: { display_name } }] }`
- Map to: `{ title, url: link, score, answer_count, is_answered, tags, asked: creation_date, asker: owner.display_name }`

Module structure (mirror `agent/community_assistant/platform_client.py` for the httpx client):
```python
_GITHUB_CLIENT = httpx.Client(base_url="https://api.github.com", timeout=10.0)
_DEVTO_CLIENT = httpx.Client(base_url="https://dev.to", timeout=10.0)
_STACKOVERFLOW_CLIENT = httpx.Client(base_url="https://api.stackexchange.com", timeout=10.0)
```

Or a single shared `httpx.Client` without base_url and full URLs at call sites — pick one approach consistently. The per-client form is preferred because it documents the upstream and lets tests `monkeypatch` each client independently.

Existing test pattern for httpx mocking (used in test_platform_client.py): `monkeypatch.setattr(mod._client, "get", _raise)` for the connection-error path. For SUCCESS responses with realistic JSON, use `httpx.MockTransport` (already referenced in agent README — `tests use fixtures from tests/fixtures/ and mock httpx via MockTransport in tests/conftest.py — no real network calls`). Check `agent/tests/conftest.py` — if MockTransport helpers exist add to them; if not, mock via `monkeypatch.setattr(<client>, "get", lambda url, **kw: httpx.Response(200, json=fixture))`.

Cloud Run deploy reference: `agent/discord_bot/README.md` documents the exact `gcloud run deploy cwa-assistant-bot --source=. --dockerfile=agent/discord_bot/Dockerfile --region=us-central1 --min-instances=1 --max-instances=1 --cpu=1 --memory=512Mi --timeout=3600 --no-allow-unauthenticated --set-secrets="..." --project=$GOOGLE_CLOUD_PROJECT` command. The Discord bot embeds the ADK Runner in-process — redeploying the bot IS redeploying the agent. The bot does NOT need `GHOST_CONTENT_API_KEY` or `YT_API_KEY` because those routes live on Vercel and the bot calls them via `PLATFORM_API_BASE_URL`. The bot DOES need `GITHUB_TOKEN` (optional) since the external_knowledge_agent calls GitHub directly from Python.
