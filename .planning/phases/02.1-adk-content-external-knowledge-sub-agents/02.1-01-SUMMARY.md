# Phase 02.1 Plan 01: Ghost Blog Search Route + content_agent Summary

Ghost blog search route and ADK content_agent sub-agent that surfaces blog.codewithahsan.dev posts in Discord via ISR-cached Next.js proxy.

## What Was Built

**Task 1 — /api/content/blog/search route (Next.js)**

- `src/app/api/content/blog/search/route.ts`: GET handler that proxies the Ghost Content API with `next: { revalidate: 3600, tags: ["content:blog"] }` ISR cache directive. Searches both `title` and `excerpt` fields via Ghost's `filter` param. Maps each post to a canonical `https://blog.codewithahsan.dev/{slug}/` URL. Handles all error paths (missing q → 400, unconfigured key → 500, upstream non-2xx → 502, fetch throw → 500).
- `src/app/api/content/blog/search/__tests__/route.test.ts`: 5 Vitest cases using `vi.stubGlobal("fetch", ...)` — no hoisting needed. Covers happy path (asserts shaped URL + ISR options), 400, 500 (key), 502, and 500 (throw).

**Task 2 — ADK content_agent + platform_client + root wiring**

- `agent/community_assistant/platform_client.py`: Added `fetch_blog_posts(query)` after `fetch_roadmaps` — single-line `_get` call, matches existing helper style.
- `agent/community_assistant/sub_agents/content_agent.py`: `_shape_blog_post` with url passthrough/fallback/None logic. `search_blog_posts(query)` tool with empty-query guard, error contract `{"status": "error", "posts": []}`, success contract `{"status": "success", "count": N, "posts": [...]}`. `content_agent = LlmAgent(name="content_agent", ...)` with instruction mandating URL citation.
- `agent/community_assistant/agent.py`: Import added, `content_agent` appended to `sub_agents`, delegation bullet added to `ROOT_INSTRUCTION`.
- `agent/tests/fixtures/content_blog.json`: 2 posts (full fields + minimal fields) with realistic data.
- `agent/tests/conftest.py`: `content_blog_payload` fixture added.
- `agent/tests/test_content_agent_blog.py`: 6 pytest cases.

## Decisions Made

1. **Ghost filter syntax**: Used comma-separated OR filter `title:~'{q}',excerpt:~'{q}'` so both title and excerpt are searched — increases recall for natural-language Discord queries.
2. **ISR tag**: Used `tags: ["content:blog"]` to enable future on-demand revalidation endpoint without code changes.
3. **feature_image + reading_time**: Included in route shape and fixture — enriches future Discord replies; consistent with plan extension for YouTube in Plan 02.
4. **`_shape_blog_post` url logic**: Raw `url` is passed through (the route already produces canonical URLs); fallback to slug-derived URL; `None` if neither — mirrors `_shape_mentor` None-not-string pattern preventing broken links.
5. **No vitest hoisting needed**: `vi.stubGlobal("fetch", ...)` approach in `beforeEach` is clean enough without shared hoisted mock variables.
