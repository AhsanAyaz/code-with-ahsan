<objective>
Link polish across all three live-data sub-agents. Every entity returned to the LLM gains an absolute `https://codewithahsan.dev/...` URL so the final Discord reply contains clickable links that drive traffic back to the platform — the Priority #3 deliverable from CONTEXT.md.

Pure Python refactor. Zero new external dependencies. Zero changes to Next.js side. The lowest-risk plan in the phase, which is exactly why it ships first: validates the test scaffolding from plan 00 with a small surface area before plans 02 and 03 introduce real complexity.

Purpose: Make the demo links clickable. Without this, the LLM either invents URLs (hallucination risk) or omits them entirely. Closing the loop "click project → see creator's mentor profile → book a session" is the conversion story we're showing on YouTube.

Output: 4 modified Python files + 4 new test files. All `<verify>` commands run in <10s against the agent/.venv with no network or Firebase access required.
</objective>
