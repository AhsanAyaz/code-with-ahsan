**Short-circuit semantics:**
- Both `before_agent_callback` and `after_agent_callback`: return `None` for side-effects only (logging). Returning a non-None `types.Content` SHORT-CIRCUITS the agent body — we never want that for telemetry, so always return `None`.

**Sync vs async (RESEARCH §3.1):** ADK awaits via `inspect.isawaitable(result)`. Plan 07-03 uses `def` (sync) — the callbacks do only string operations + a `sys.stdout.write/flush`, no I/O. Sync gives cleaner stack traces. (Plan 07-02's tool cache callbacks use `async def` because they read `datetime.now` and the cache call site sits inside Discord's asyncio event loop; the lifecycle callbacks could be either — sync is simpler and behaves identically.)

**Timing state-key convention (RESEARCH §6 + §9 Pitfall 5):**
- `before_agent_callback` writes `_cb_enter_{agent_name}` to state with `int(datetime.now(timezone.utc).timestamp() * 1000)` (epoch ms).
- `after_agent_callback` reads the same key, computes `duration_ms = now_ms - enter_ms`.
- Private key prefix `_cb_` prevents collision with output_key namespace (RESEARCH §6 line 330).
- For parallel branches (gh/devto/so under ExternalFanOut): each branch has a DISTINCT agent_name, so DISTINCT state keys — no race (RESEARCH §9 Pitfall 5).

**HMAC session_id helper (RESEARCH §11 verbatim, A4 decision: A1 direct-env-read):**
```python
def _hmac_session_id(session_id: str) -> str:
    import os
    secret = os.environ.get("USAGE_HASH_SECRET", "")
    if not session_id or not secret:
        return ""
    import hashlib, hmac
    mac = hmac.new(secret.encode("utf-8"), session_id.encode("utf-8"), hashlib.sha256)
    return mac.hexdigest()[:16]
```
**Why direct-env-read (RESEARCH §15 A4 decision):** the factory pattern (`make_lifecycle_callbacks(secret)`) requires deferring agent construction to a wiring function, but agents are currently instantiated at module-import time. Direct env-read at callback-call time has the same degradation path as `bot.py`'s existing HMAC convention — if `USAGE_HASH_SECRET` is unset, `session_id_hash` is `""` (no crash, no PII leak, just lost cross-turn correlation). Cloud Run env vars are set BEFORE Python imports run, so the env var is always present when the first callback fires (RESEARCH §15 A4).

**Alternative — reuse `discord_bot.usage_metrics.hash_user_id`:** The existing helper at `agent/discord_bot/usage_metrics.py:31-40` does the exact same HMAC pattern. Plan 07-03 *could* `from discord_bot.usage_metrics import hash_user_id` and call `hash_user_id(session_id, os.environ.get("USAGE_HASH_SECRET", ""))` to co-locate the HMAC convention. **Decision (conditional, see Task 2a Step A.1 probe):** default path is to REUSE Plan 07-01's module-local `_hmac_session_id` helper directly (no second import dependency). The cross-package `discord_bot.usage_metrics.hash_user_id` swap is an OPT-IN path gated on the import-chain probe in Task 2a Step A.1 — only enabled if the probe confirms no circular import risk from the `agent.community_assistant` → `discord_bot` direction.
