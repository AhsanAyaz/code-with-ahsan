Callback signature (RESEARCH §3.2 VERIFIED from llm_agent.py):

**Short-circuit semantics:** return `None` → let model call proceed (mutate `llm_request.contents[-1].parts[0].text` in place for redaction). Return `LlmResponse` → skip model call entirely (NOT what we want).

**Why root_agent ONLY:**
- RESEARCH §3.2 + §15 A1 (VERIFIED): `before_model_callback` on `root_agent` fires only for `root_agent`'s own LLM call. Sub-agents have their own `_run_async_impl` paths. **Mutation pattern (RESEARCH §11 verified):**
- Iterate `llm_request.contents` in `reversed(...)` until the most recent `role == "user"` content with non-empty `parts`.
- Run each `PII_PATTERNS[*].sub(REDACTION_PLACEHOLDER, text)` and accumulate redacted type labels in a list.
- If any redactions: assign back to `content.parts[0].text` AND emit a single structured stdout JSON event.
- `break` after processing the most recent user message — never walk further back into model replies.

**PII pattern catalog (RESEARCH §4 VERIFIED against example_02):**
- `CREDIT_CARD`: `r"\b(?:\d[ -]?){13,16}\b"` — 13-16 digits with optional separator.

**HMAC session_id helper (INLINE in callbacks.py — no import from `discord_bot.usage_metrics`):**
```python
def _hmac_session_id(session_id: str) -> str:
    """HMAC-SHA256 of session_id keyed by USAGE_HASH_SECRET env var, truncated to 16 hex chars.
    Returns "" when either input is missing (matches usage_metrics.hash_user_id degradation path).
    Inline implementation (not imported) — avoids circular-import risk RESEARCH §11 flags:
    `discord_bot.usage_metrics` indirectly imports `community_assistant` via `bot.py`,
    and `callbacks.py` is imported by ~11 sub-agent files. Plans 07-02 and 07-03 REUSE
    this helper directly from the same module — no extraction needed."""
    import hashlib, hmac, os
    secret = os.environ.get("USAGE_HASH_SECRET", "")
    if not session_id or not secret:
        return ""
    mac = hmac.new(secret.encode("utf-8"), session_id.encode("utf-8"), hashlib.sha256)
    return mac.hexdigest()[:16]
```

The `callback_context.session.id` (UUID) is the raw input; the HMAC'd 16-hex form is what appears in the event payload. Raw session_id NEVER serialized to the event.
