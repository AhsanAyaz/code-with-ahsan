2. Create `src/app/api/stats/route.ts` — Public GET endpoint (no auth required). It must:
   - Query Firestore for: mentor count (mentorship_profiles where role=='mentor' AND status=='accepted'), mentee count (mentorship_profiles where role=='mentee'), active mentorships (mentorship_sessions where status=='active'), completed mentorships (mentorship_sessions where status=='completed'), average rating (from mentor_ratings collection).
   - Import and include socialReach config in the response.
   - Implement in-memory caching: store the computed stats in a module-level variable with a timestamp. If the cache is less than 5 minutes old, return cached data instead of querying Firestore. Use `Cache-Control: public, s-maxage=300, stale-while-revalidate=60` header on the response.
   - Return JSON shape:
     ```json
     {
       "community": {
         "mentors": number,
         "mentees": number,
         "activeMentorships": number,
         "completedMentorships": number,
         "averageRating": number
       },
       "social": { ... socialReach object },
       "cachedAt": "ISO string"
     }
     ```
   - Wrap in try/catch, return 500 with `{ error: "Failed to fetch stats" }` on failure.
   - IMPORTANT: For mentor count, filter by status=='accepted' (not just role=='mentor') to only count active mentors. Reference the existing admin stats route for query patterns but simplify — no alerts, no pending counts, no admin-only data.
