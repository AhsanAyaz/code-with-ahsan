**Create `src/app/admin/courses/page.tsx`** as a `"use client"` component with these sections:

**1. Course List (top section):**
- On mount, `GET /api/admin/courses` with admin token header
- Display courses in a table (DaisyUI `table` class): columns — Name, Slug, Chapters, Posts, Published Date, Actions
- Actions column: Delete button (DaisyUI `btn btn-error btn-sm`)
- Delete shows a confirm dialog (`window.confirm("Delete course {name}? This removes all MDX files.")`) then calls `DELETE /api/admin/courses/{slug}`
- After delete, refetch the list
- Show loading spinner while fetching (DaisyUI `loading loading-spinner`)
- Show "No courses found" if empty

**2. Add Course Form (below list, in a DaisyUI `card`):**
- Title: "Add New Course"
- Step 1 — Input mode selector: two DaisyUI radio buttons — "YouTube Video" or "YouTube Playlist"
- Step 2 — YouTube ID input field + "Fetch" button
  - For video mode: text input for video ID (e.g., "oUmVFHlwZsI"), fetch button calls `GET /api/admin/courses/youtube?videoId=xxx`
  - For playlist mode: text input for playlist ID, fetch button calls `GET /api/admin/courses/youtube?playlistId=xxx`
- Step 3 — After fetch, display preview:
  - For video: Show video title, thumbnail, and parsed chapters list (title + timestamp). Allow editing chapter titles. Show editable fields for course name (pre-filled from video title), slug (auto-generated from name using kebab-case), description.
  - For playlist: Show list of videos. Allow editing course name and slug. Each video becomes a post.
- Step 4 — "Create Course" button
  - For video mode: POST to `/api/admin/courses` with `{ slug, name, description, outline, videoId, chapters }`
  - For playlist mode: POST to `/api/admin/courses` with `{ slug, name, description, outline, videoId: items[0].videoId, chapters }` where chapters are derived from playlist items (each item = a chapter with timestampSeconds=0, using the individual video URLs)
  - On success: show toast ("Course created!"), reset form, refetch course list
  - On error: show toast with error message

**State management:** Use React useState for: courses array, loading, input mode, youtubeId, fetchedData, form fields (name, slug, description), creating state.

**Slug generation:** When user types course name, auto-generate slug: lowercase, replace spaces/special chars with hyphens, remove consecutive hyphens.

**Style:** Match existing admin pages — use DaisyUI components (card, table, btn, input, radio, badge). Keep consistent with the admin layout's `max-w-7xl mx-auto` container.

**Toast:** Use `useToast()` from `@/contexts/ToastContext` (same as other admin pages).

**Admin token:** Get from `localStorage.getItem(ADMIN_TOKEN_KEY)` and pass as `x-admin-token` header on all fetch calls.
