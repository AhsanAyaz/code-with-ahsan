<task type="auto">
  <name>Task 1: Create course MDX utility library and YouTube API route</name>
  <files>src/lib/course-mdx.ts, src/app/api/admin/courses/youtube/route.ts</files>
  <action>
Create `src/lib/course-mdx.ts` with these functions:

1. `listCourses()` — Read all directories in `src/content/courses/`, parse each `course.mdx` frontmatter using `gray-matter`, return array of `{ slug, name, description, chapters: number, posts: number, publishedAt }`. Count posts by reading the `posts/` subdirectory. Sort by name.

2. `createCourse(data)` — Accepts `{ slug, name, description, outline, videoId, chapters }` where chapters is `Array<{ title: string, timestampSeconds: number }>`. Creates:
   - Directory `src/content/courses/{slug}/posts/`
   - `course.mdx` with frontmatter matching existing format (generate incremental `id` based on max existing id + 1, set `publishedAt` to now, single chapter with `id` = max existing chapter id + 1, `visibilityOrder` = max existing + 1, default author "Muhammad Ahsan Ayaz")
   - For each chapter timestamp: create a post MDX file in `posts/` with frontmatter matching existing format. The `videoUrl` should be `https://www.youtube.com/watch?v={videoId}&t={timestampSeconds}`. Post `id` should be incremental from max existing post id. File naming: `000-{order:3}-{slugified-title}.mdx`. Use `github-slugger` (already in deps) for slug generation.

3. `deleteCourse(slug)` — Validate slug exists in `src/content/courses/`, then `fs.rmSync` the directory recursively. Return success/failure.

4. `getMaxIds()` — Helper that scans all existing course/chapter/post MDX files to find the current max id, max chapter id, and max post id (needed for auto-incrementing).

Use `path.join(process.cwd(), 'src/content/courses')` as the root. Import `gray-matter` for parsing/writing MDX. Import `fs` and `path`. Use `matter.stringify()` for creating MDX content (same pattern as convert-course-json-to-mdx.js).

Create `src/app/api/admin/courses/youtube/route.ts`:

- `GET` handler accepting query params `?videoId=xxx` or `?playlistId=xxx`
- For `videoId`: Use `googleapis` YouTube Data API v3 to fetch:
  - `videos.list({ part: ['snippet', 'contentDetails'], id: [videoId] })` for title, description, thumbnail
  - Parse the video description for chapter timestamps. YouTube chapters appear in description as lines matching pattern `MM:SS Title` or `H:MM:SS Title` (e.g., "0:00 Introduction", "5:30 Setting Up"). Parse these into `{ title, timestampSeconds }` array.
- For `playlistId`: Use `playlistItems.list({ part: ['snippet', 'contentDetails'], playlistId, maxResults: 50 })` to fetch all videos in the playlist. Return array of `{ videoId, title, position }`.
- Auth: Check `x-admin-token` header (import ADMIN_TOKEN_KEY pattern — check against `process.env.ADMIN_TOKEN`). Return 401 if missing/invalid.
- Return JSON response with the fetched data.

Important: Use `process.env.YT_API_KEY` for YouTube API auth (already configured in .env).
